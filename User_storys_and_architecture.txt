Проект по разраотка на софтуер:

git repository: https://github.com/dahlia-d/library-web-back-end-API.git

User storys:
	- Потребителя може да се регистрира и да се логва в акаунт
		Акаунта има име на потребителя, парола, email
	- Потребителя може да търси книга от библиотеката
		Книгата има заглавие, автор, описание
	- Потребителя може да види дали една книга е свободна
		Книга има и статус - заета или незаета
	- Потреителя може да направи заявка за взимане на книга - потребителите трябва да могат да направят заявка за взимане на книгата, ако книгата е свободна, тя се запазва за съответния потребител, ако не е потребителя просто не може да я запази ( или се слага в лист за чакащи за книгата, и когато се освободи потребителя се известява ).
		Книгата се пази за съответния потребител няколко дни, преди отново да бъде отбелязана като свободна.

Aрхитектура:
	- Databases access
	- Models:
		- User model
		- Book model
	- Controllers:
		- User controller
		- Book controller
		- API controller
	- Services:
		- User service
		- Book service
